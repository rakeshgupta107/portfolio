Mercury: Building a Unified Platform for 400+ Global Events

Led the build of Mercury, RX Global's unified event platform—consolidating registration, ecommerce, badging, access control, and customer service into one integrated system. Now powering hundreds of events attended by millions worldwide.

400+ Events powered globally
Millions Annual attendees
15+ Regions worldwide
5 Core platform modules

The Challenge

RX Global (formerly Reed Exhibitions) produces over 400 events annually across 15+ regions—from small trade shows to massive exhibitions with hundreds of thousands of attendees. Each event required registration, badging, access control, exhibitor commerce, and customer support.

The existing approach relied on fragmented third-party tools cobbled together differently for each event. Registration might use one vendor, badging another, ecommerce a third. This created:

  • Inconsistent attendee experience: Different registration flows, unclear processes, frequent handoffs
  • Operational complexity: Event teams managing multiple vendors, duplicate data entry, manual reconciliation
  • Limited visibility: No unified view of attendees, purchases, or event performance
  • Integration nightmares: Each tool had its own data model, API, and support process
  • Scaling issues: Adding new events meant procurement, setup, training across multiple systems

The opportunity: Build a unified platform that brought registration, ecommerce, badging, access control, and customer service into one integrated system—streamlining operations and improving experience for both attendees and event teams.

The Mercury Platform

Mercury was designed as an integrated platform with five core modules, each addressing a critical part of the event lifecycle:

Mercury's five integrated modules: Registration, eCommerce, Badging & Access Control, Reporting, and Customer Services—all working from a single data foundation

Rather than separate tools that needed integration, Mercury was built as a unified system from the start. A single attendee record flowed through the entire journey—from initial registration through onsite check-in, with purchases, badges, and support interactions all connected.

1. Registration: Foundation of Every Event

Mercury allowed event teams to set up tailored registration flows, manage real-time attendee data, and streamline onboarding across different event types and regions.

Multi-step registration flow: Attendee details → Company information → Order summary with real-time validation and progress tracking

Key capabilities:

  • Flexible form builder supporting various event types (conferences, trade shows, exhibitions)
  • Multi-step flows with conditional logic based on attendee type or selections
  • Real-time data validation and duplicate detection
  • Support for individual, group, and corporate registrations
  • Multi-language and multi-currency support for global events
Complete registration journey from initial landing page through confirmation—designed for conversion and clarity across all event types

2. Badging System: Automated & Flexible

Event teams could manage badge designs, layouts, and printing rules centrally. Badges were automatically issued on confirmation or at check-in, reducing manual steps and errors.

Automated badge generation from registration confirmation to physical badge with QR code for access control and tracking

Features:

  • Template-based badge design system with drag-and-drop layout
  • Dynamic content (name, company, badge type, QR codes) populated from registration
  • Print queue management for on-site badge production
  • Pre-printed badge options with variable data for VIPs and exhibitors
  • Integrated with access control via unique QR codes and identifiers

3. Access Control & Onsite Operations

Tightly linked with registration and badging, access control allowed organizers to define entry rules by attendee type or badge scan, with real-time insights into venue capacity and zone access.

Onsite access control: Handheld scanner → Badge validation → Entry confirmation with real-time verification against attendee permissions

Onsite capabilities:

  • Mobile scanning apps for fast check-in at entrances and session rooms
  • Zone-based access rules (main show floor, VIP lounges, conference sessions)
  • Real-time attendance tracking and capacity monitoring
  • Offline mode for reliable operation even with connectivity issues
Registration manager dashboard showing real-time performance statistics: registrations by day, attendance patterns, and badge type breakdowns—enabling data-driven event management

4. Exhibitor eCommerce: Self-Service Platform

Exhibitors could browse services, book packages, manage logistics, and track payments—all within one intuitive interface. This shifted the exhibitor experience from phone calls and spreadsheets to modern self-service.

Exhibitor dashboard (left) and package selection (right)—from company profile management to stand selection, exhibitors control their event presence

Exhibitor capabilities:

  • Package browsing and comparison (stand sizes, locations, included services)
  • Add-on services (power, WiFi, furniture, catering, marketing opportunities)
  • Stand location selection with interactive floor plans
  • Company profile management (logo, description, products, staff)
  • Payment tracking and invoice management
Complete exhibitor purchase flow: Package selection → Review and checkout → Order confirmation → Automated email confirmation—streamlining what was previously a manual, multi-week process

Every service booked flowed directly into the platform, with order history, item breakdowns, and payment confirmation instantly available to both exhibitors and event operations teams.

5. Customer Services: Unified Support View

Support teams gained visibility into each attendee's complete journey—registrations, purchases, badge status, and onsite interactions. This meant fewer handoffs and faster resolution.

Customer service search and detail views—staff can quickly look up any attendee and see their complete event history, enabling fast, accurate support onsite or remotely

Support capabilities:

  • Unified attendee search across all events and registration types
  • Complete history view: registrations, purchases, badge prints, access scans
  • Quick actions: resend confirmations, reprint badges, update details
  • Issue tracking and resolution workflows
  • Especially valuable onsite where speed and accuracy matter most

Automated Communications Throughout

Confirmation emails were triggered at each key step—registration completion, order placement, badge issuance, payment confirmation. This improved clarity, reduced support queries, and helped attendees feel in control of their journey.

Email templates were customizable per event while maintaining consistent branding and information architecture across all communications.

What I Learned

1. Platform Products Require Different Thinking Than Features

Building Mercury wasn't about delivering features—it was about creating infrastructure that would scale across hundreds of use cases. Every decision had to consider: Will this work for a 200-person conference AND a 100,000-person exhibition? This required different prioritization and design thinking.

2. Integration is Product Strategy, Not Technical Detail

The real value of Mercury wasn't individual modules—it was how they worked together. A registration flowing seamlessly to badging to access control to support. The connections between modules mattered more than any single feature. This shaped how we designed APIs, data models, and user workflows.

3. Replacing Existing Tools is Harder Than Building New Ones

Event teams had years of muscle memory with their existing (fragmented) tools. Mercury had to be clearly better AND easy to migrate to. We couldn't just match existing functionality—we had to deliver enough additional value to justify the switching cost.

4. Global Scale Reveals Assumptions

What worked for UK events didn't always work in Asia, Latin America, or Middle East. Time zones, currencies, languages, payment methods, cultural expectations—all had to be considered. Building for global scale from the start (not bolting it on later) was essential.

5. Onsite Performance is Non-Negotiable

When 5,000 people arrive at an exhibition at 8am, the system HAS to work. No "try again later," no graceful degradation. This drove our approach to reliability, offline capabilities, and load testing. Event day is when platform investments pay off or fail publicly.
