Mercury: A Unified Platform for Registration, Commerce, and Onsite Operations

Mercury is RX Global’s core platform for registration, ecommerce, access control, and on-site operations across large-scale events worldwide. I led product definition and system design to consolidate a fragmented vendor landscape into a single, integrated platform supporting complex, revenue-critical workflows.

Critical Time-sensitive event workflows
Unified Single attendee data model
Reusable Config-driven event setups
Resilient Onsite + offline operation

Context & problem

  • Core event workflows were split across multiple vendors and tools, creating inconsistent data and duplicated effort
  • Operational teams needed predictable behaviour during high-traffic, time-critical onsite periods
  • Fragmentation limited reuse across 400+ events and increased operational risk
  • The platform had to support varied event configurations without becoming fragile or bespoke

Constraints & realities (what this required)

  • Reliability under pressure: onsite operations need predictable behaviour and fast recovery paths
  • Configuration complexity: events vary widely; workflows must be flexible without becoming bespoke
  • Integration surface: payments, access hardware, comms, and third-party services need clean interfaces
  • Failure modes: graceful degradation is a feature, not an edge case

System design & approach

Mercury consolidated critical event capabilities into a shared platform with reusable services and configuration-driven workflows. The goal was operational reliability first: consistent data, predictable behaviour under load, and clear failure handling across the event lifecycle.

The Mercury Platform

Mercury was designed as an integrated platform with five core modules, each addressing a critical part of the event lifecycle.

Mercury's five integrated modules: Registration, eCommerce, Badging & Access Control, Reporting, and Customer Services—all working from a single data foundation

Rather than separate tools that needed integration, Mercury was built as a unified system from the start. A single attendee record flowed through the entire journey—from initial registration through onsite check-in, with purchases, badges, and support interactions all connected.

Outcomes

  • Replaced a fragmented vendor landscape with a unified platform supporting registration, commerce, and onsite operations
  • Improved operational reliability for high-traffic, time-critical event scenarios
  • Enabled reuse of core capabilities across hundreds of events with configuration-driven workflows
  • Created a consistent data foundation for reporting, support, and downstream integrations
